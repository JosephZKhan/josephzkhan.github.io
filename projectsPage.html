The Bonehead Battle

Online Crowd Multiplayer for Public Spaces
February - April 2025



This game was made for a module on games technology for public spaces. It demonstrates a form of seamless drop in/out online multiplayer supporting a flexible player count.

Players connect to the game by accessing a webpage on their smart device via the displayed QR code. This webpage has buttons emulating a gamepad, controlling an avatar that is spawned in once a username is entered.

I have a soft spot for bullet-hell games like Cuphead, Enter The Gungeon and Vampire Survivors, which influenced the direction of the projectile patterns in the boss battle gameplay.

WebSockets were used to establish a connection between the web client and Unity server, and transfer button presses from different clients into input signals.

Procedural Dungeon Generation Framework

Preserving Human Authorship Within Procedurally Generated Levels
October 2024 - April 2025



This level generator was made for my dissertation.

This project had two main goals for procedural generation: Making a generator whose output appears hand-built, and making an interface for a human designer to specify designs for a generator in the most detailed and effective method possible.

A major technical accomplishment was the implementation of a generative grammar structure for settings data.

Inspired by this breakdown of Enter The Gungeon's level generation, this format of modular, node-based settings data per-room lets a human express designs in the vocabulary of game content.

The project utilised Unity's Tilemap support for level geometry. Custom editor scripts created a bespoke display for generation settings, making actions such as assigning references between rooms simpler and easier to perform. Click here to view this repository.

The generator is set up in the Scene called LevelGenerationTestScene. To control the character, use arrows or WASD to move and Spacebar to shoot.

Block Buster Bonanza

Voice Commands For Inclusivity
October 2024 - January 2025



For a module on games technology for inclusivity, I built this to demonstrate how speech input can be utilised to achieve high-reflex gameplay with input combos typically demanding physical dexterity to perform.

This was made in Unity, with Microsoft Azure's Speech SDK for the speech recognition. It was a fun experiment with async operations, required to run the continuous speech input system.

To fully leverage the inclusivity benefits of the speech input method, settings were implemented allowing a user to set their own input terms via recording. The settings data is saved externally and loaded in on launch, for ease of use.

81 different speech regions are supported, to broaden the range of usability.

Reaction-Diffusion Effect
2021

This was an attempt to recreate the swirling blood effect found on the death screen of Control (2019). After learning about Reaction-Diffusion, the mathematical model used to represent differently-coloured chemicals diffusing into one another, I found a way to implement this using multiple shader files updating and reading from a custom render target. The resultant effect shows a pattern spreading across a surface. The visible properties and diffusion rates of the two 'chemicals' can be customized, and the effect works on any mesh, allowing for a variety of uses.
